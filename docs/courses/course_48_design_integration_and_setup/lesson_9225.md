# Исходящая почта через Яндекс

Ранее в **Яндекс.Почте** можно было авторизоваться под одним логином, а отправлять письма от имени другого пользователя, используя почтовый сервер **smtp.yandex.ru.**



Но в связи с участившимися случаями фишинга почтовый сервис **Яндекс.Почта** включил строгий контроль адреса отправителя. Вы больше не сможете через SMTP-клиент отправить письмо, если отправитель в поле **From** («От кого») отличается от авторизованного пользователя. Указать несколько отправителей в поле **From** также не получится.



## Что это значит?



В коробочной версии *«Битрикс24»* исходящая почта настраивается на уровне сервера.



Если вы использовали бесплатный почтовый сервис **Яндекс.Почта** для отправки писем из вашего *«Битрикс24»* несколькими пользователями, у которых обратная почта отличается (заголовок письма From) от почты для smtp-авторизации, то теперь такие письма отправляться не будут. В логах будут ошибки вида `Sender or From header address rejected: not owned by authorized user`.




## Что делать?



Есть несколько вариантов решения:



1. Отправлять письма с одного e-mail, который указан в настройках почтового клиента. В **VMBitrix** это **msmtp**.
2. Использовать сервер почты вашего хостинга. Для этого обратитесь в техническую поддержку хостинга.
3. Арендовать почтовый сервер или настроить свой собственный.
4. Установить подходящее приложение из **Битрикс24.Маркет**.



**Примечание.** Для настройки локального почтового сервера потребуется привлечение программистов и администратора сервера. Вы также можете воспользоваться услугами [наших партнеров](https://www.bitrix24.ru/partners/).
