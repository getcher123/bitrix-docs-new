| --- | --- |




Функционал резервных кодов используется для доступа на сайт в тех случаях, когда нет возможности использовать устройство двухэтапной авторизации. Например, если утерян брелок или телефон с приложением.




**Примечание:** Доступность функционала резервных кодов зависит от

			настроек двухэтапной авторизации.


На закладке **Параметры** (Настройки &gt; Проактивная защита &gt; Двухэтапная авторизация) можно настроить параметры механизма двухэтапной авторизации.

[Подробнее](lesson_5002.md#otp_settings)...





Администратор сайта может получить список резервных кодов с помощью ссылки **Резервные коды** на странице

			редактирования пользователя


Управление учетными записями пользователей в системе выполняется на странице **Список пользователей** (Настройки &gt; Пользователи &gt; Список пользователей).


Форма редактирование параметров пользователя вызывается с помощью двойного клика левой кнопкой мыши по записи пользователя, либо с помощью пункта **Изменить** в контекстном меню. С помощью команды **Авторизоваться** администратор может в один клик выйти из своего аккаунта и открыть форму авторизации под выбранным аккаунтом.

[Подробнее](lesson_2004.md)...

		 (Настройки &gt; Пользователи &gt; Список пользователей), вкладка **Двухэтапная авторизация**:



![](../../../images/courses/48/dev.1c-bitrix.ru/images/admin_expert/security/otp/user_otp_recovery_codes.png)




Действительны только коды, созданные последними.




Также, в случае, когда пользователь не может воспользоваться своим устройством для входа на сайт, можно просто на время

			отключить двухэтапную авторизацию

                    	![](../../../images/courses/48/dev.1c-bitrix.ru/images/admin_expert/security/otp/user_otp_enabled.png)


**Отключить** - позволяет отключить двухэтапную авторизацию **бессрочно** или же на **определенное** количество дней.

[Подробнее](lesson_5005.md#otp_off)...

		 для него.




**Примечание**: В публичной части сайта получение **Резервных кодов** пользователем можно сделать с помощью создания администратором сайта страницы и размещения на ней

			компонента Резервные коды OTP


В редакциях **Стандарт** и **Малый бизнес** есть модуль **Проактивная защита**, но нет модуля **Социальная сеть**, которая выводит форму для подключения пользователями двухэтапной авторизации. Администратор должен создать публичные страницы для размещения компонентов подключения и резервных кодов.


Для вывода формы инициализации подключения нового устройства двухэтапной авторизации в публичной части используется компонент **Подключение устройства OTP** .
