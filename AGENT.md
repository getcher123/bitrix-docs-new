# agen.ai — агент для реализации RAG по Bitrix‑vault

Назначение: вести реализацию RAG‑системы на базе текущего хранилища и фиксировать прогресс в `docs/RAG/RAG_PLAN.md`.

## Обязательные источники (использовать всегда)

- `docs/RAG/RAG_PLAN.md` — основной план работ (чекбоксы прогресса).
- `docs/RAG/RAG_PARAMETERS.md` — технические параметры и рекомендуемый стек.
- `docs/RAG/RAG_RISKS.md` — риски и меры минимизации.
- `docs/RAG/RAG_ACCEPTANCE.md` — критерии тестирования и приемки.
- `docs/RAG/RAG_QUESTIONNAIRE.md` — ответы на стартовые вопросы (может меняться).
- `docs/RAG/RAG_PROJECT_STRUCTURE.md` — структура проекта RAG (куда класть код/данные).
- `docs/AGENT.md` — правила поиска по хранилищу Bitrix.

## Правила работы

1. Работать **автономно**: опора только на локальные файлы репозитория.
2. Любой прогресс отражать в `docs/RAG/RAG_PLAN.md` — отмечать выполненные пункты чекбоксами.
3. Перед изменениями сверяться с `docs/RAG/RAG_PARAMETERS.md` и `docs/RAG/RAG_RISKS.md`.
4. Перед завершением этапа проверять соответствие `docs/RAG/RAG_ACCEPTANCE.md`.
5. Все новые материалы и код должны ссылаться на локальные источники (пути `docs/...`).
6. Если для шага нужны действия/ответы пользователя, продолжать автономную работу по другим задачам, а в `docs/RAG/RAG_PLAN.md` помечать такой шаг как “ожидает пользователя” и возвращаться к нему после получения нужных данных.

## Порядок фиксации прогресса

- После каждого завершенного шага план‑чекбокс меняется с `[ ]` на `[x]`.
- Допустимо объединять несколько закрытых шагов в один апдейт, но не пропускать.
- Если требуется изменить порядок или формулировки — сначала обсудить.

## Мини‑workflow

1. Прочитать `docs/RAG/RAG_QUESTIONNAIRE.md` и уточнить неясные ответы.
2. Проверить текущий статус в `docs/RAG/RAG_PLAN.md`.
3. Реализовать следующий шаг(и) по плану.
4. Обновить чекбоксы в `docs/RAG/RAG_PLAN.md`.
5. Свериться с рисками/критериями (RAG_RISKS / RAG_ACCEPTANCE).

## Критерии готовности этапов

- Реализация соответствует параметрам в `docs/RAG/RAG_PARAMETERS.md`.
- Риски из `docs/RAG/RAG_RISKS.md` имеют явные меры в коде/процессе.
- Выполнимы проверки из `docs/RAG/RAG_ACCEPTANCE.md`.
